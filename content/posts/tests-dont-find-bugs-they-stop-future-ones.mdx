As a beginner I thought tests were for finding bugs....

Feature works → ship it. why write more code to check code that already runs 🤷‍♂️

then one day i fixed a small bug in one place.... and broke something completely unrelated somewhere else. didn't even notice for a while 🥲

a test would have caught it the second i made the change.

That i when it clicked.... tests are not for finding bugs today. the code already works today. tests are for protecting that working code when future-me changes something months later.

<Ex>Ex: extract a string, rename a function, refactor a flow.... a good test tells you instantly if you just broke something.</Ex>

<Pull>tests don't find bugs. they stop future bugs from shipping 😮‍💨</Pull>
