small bug in faculty enrollment. gender is marked mandatory, but the form let you skip it and move to the next page anyway 🤷‍♂️

i asked claude to fix it. it fixed the frontend, then said the backend needed a guard too — and it was right about that part. the data layer had no check at all. an empty gender could walk straight into the DB.

so far so good. then i actually looked at what it wrote.

```text
the AI's rule:
  body('gender').isString().notEmpty().isLength({ max: 20 })

the field on screen:
  a dropdown.  Male / Female / Other.  three options. that's it.
```

a length check. on a dropdown 🥲

think about what `isLength({ max: 20 })` actually allows. it accepts "banana". it accepts "asdfgh". it accepts any 20 characters of nonsense. it was never guarding the field.... it was guarding the field's *length*. two completely different things.

the right rule is an allowlist:

```text
  body('gender').isIn(['Male', 'Female', 'Other'])
```

required and valid-value, in one line.

![Claude's diff in teacherValidator.ts replacing the gender length check with an isIn allowlist of Male, Female, Other](/journal/ai-fix-diff.webp)

*the actual fix in the repo — the length check (red) becomes the allowlist (green)*

here's the part that actually got me. i had ALREADY told it the options were Male / Female / Other. earlier in the same chat it read the dropdown and quoted them back to me. it had the right answer in its hands.... and still reached for a length check 😮‍💨

so i asked it straight: you're an AI agent with "top-tier intelligence" — why the silly mistake?

its own answer was the whole lesson. one word — **pattern-mimicry**.

when it went to harden the backend, it grabbed `studentValidator.ts` as the template, because that was the nearest validator that already enforced gender. and student used `isString().notEmpty().isLength({ max: 20 })`. so it just copied the *shape* of that rule.... without ever re-asking whether a length check was even the right tool for this field.

its exact words: it was in "match the neighbouring validator" mode, instead of "reason from what this field actually is" mode 😮‍💨

![Claude's own root-cause write-up calling the mistake pattern-mimicry — it copied the shape of the neighbouring student validator instead of reasoning from what the gender field actually is](/journal/ai-root-cause.webp)

*i asked for the root cause. this was its own post-mortem, word for word.*

it reasoned backwards — nearest pattern → rule. it should be:

<Pull>know the field's real values → write that rule → let the DB column be a sanity backstop, never the source of truth.</Pull>

then it hit me. if it *copied* the student validator.... then the student validator had the exact same bug. so i sent it back to fix `studentValidator.ts` too — same length check, same wrong tool, sitting in production this whole time. one bad pattern had quietly cloned itself, and the AI happily carried it forward instead of catching it.

![Claude confirming gender is a closed dropdown so the correct rule is an isIn allowlist, and noting studentValidator.ts still carried the same looser length check](/journal/ai-same-pattern.webp)

*it even admitted the student validator had the same weakness — the pattern it cloned from was wrong at the source.*

<Ex>Ex: "valid code" and "correct code" are not the same thing. the AI handed me valid. only the domain told me it was wrong.</Ex>

that's the realization that actually stuck. this is a "top-tier" model — claude 4.8 — and it didn't *think*. it pattern-matched. it saw a validator next door, copied it, and moved on. fast and confident and wrong. and honestly every AI i've used is the same.... they autocomplete from whatever's nearest. they don't *understand* your product.

the only reason this got caught is a habit my team lead **Jagan Kumar Mudila** sir drilled into me — always question AI-written code. never trust it because it looks right or because it compiles. read it like a junior's PR, every single line 🙏

the AI is a brilliant tool. it is not the person who understands your domain. that part is still my job 🤷‍♂️
