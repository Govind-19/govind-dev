i learned to code in the AI era. gemini, claude, all of it. so yes, i vibe-code. a lot 😅

and honestly it works. for UI scaffolding, boilerplate, test data, first drafts.... AI gets me 80% there crazy fast.

the problem is the 80% LOOKS finished.

it compiles. the happy path works. the demo goes well. you start believing its done.

then a senior reviews it and asks one question.

"what happens when this fails?"

<Pull>The last 20% is where production lives. Error states, edge cases, the slow query, the school with 3000 students when you tested with 30.</Pull>

i shipped AI-assisted code that worked perfectly in dev and fell over on real data. more than once 🥲

<Ex>Ex: AI wrote me a clean attendance query. nobody told it one school marks attendance for the whole month in one go. dev data: fine. that school: timeout.</Ex>

so my rule now....

vibe-code the draft. never the decision. AI writes the first version, but i have to be able to defend every line in review like i wrote it. because the reviewer doesnt care who wrote it. production doesnt either.

the seniors reviewing my PRs is what made AI safe for me to use. speed from the machine, judgment from the humans who've seen things break.

thats the combo. either one alone fails differently.
